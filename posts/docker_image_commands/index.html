Docker image - commands

Most important docker commands related to images!

Posted on 08 Apr 2020 by Łukasz Bartnicki 5 mins read Tested on: Docker 18.09
Share on:

Docker has got couple commands that allows us to manage images.

List images

Usage of docker image ls is described in another post: Docker image - list

Remove images

Let’s check what images we have locally:

[lukas@docker-host ~]# docker image ls
REPOSITORY                      TAG                 IMAGE ID            CREATED             SIZE
postgres                        latest              9907cacf0c01        8 days ago          314MB
alpine                          latest              a187dde48cd2        2 weeks ago         5.6MB
ubuntu                          18.04               4e5021d210f6        2 weeks ago         64.2MB
ubuntu                          latest              4e5021d210f6        2 weeks ago         64.2MB
ubuntu                          19.04               c88ac1f841b7        2 months ago        70MB
schickling/postgres-backup-s3   latest              e8e41fa54edb        21 months ago       117MB

We have ubuntu image that has got two aliases: ubuntu:latest and ubuntu:18.04
Let’s remove one of it:

[lukas@docker-host ~]# docker image rm ubuntu:18.04
Untagged: ubuntu:18.04

[lukas@docker-host ~]# docker image ls
REPOSITORY                      TAG                 IMAGE ID            CREATED             SIZE
postgres                        latest              9907cacf0c01        8 days ago          314MB
alpine                          latest              a187dde48cd2        2 weeks ago         5.6MB
ubuntu                          latest              4e5021d210f6        2 weeks ago         64.2MB
ubuntu                          19.04               c88ac1f841b7        2 months ago        70MB
schickling/postgres-backup-s3   latest              e8e41fa54edb        21 months ago       117MB

As we saw docker image rm removed only alias - if we remove last image alias - image will be deleted also - precisely layers that are no longer needed:

[lukas@docker-host ~]# docker image rm ubuntu:latest

Normally docker doesn’t allow us to delete image from which containers are currently running. We can use --force in remove command, it will remove also containers running from images we are trying to delete.

Remove dandling and unused images

We can remove dandling(without repository and tag) images with docker image prune. In our example all images has got repository and tag so none of them will be removed.

[lukas@docker-host ~]# docker image ls
REPOSITORY                      TAG                 IMAGE ID            CREATED             SIZE
postgres                        latest              9907cacf0c01        8 days ago          314MB
alpine                          latest              a187dde48cd2        2 weeks ago         5.6MB
ubuntu                          18.04               4e5021d210f6        2 weeks ago         64.2MB
ubuntu                          19.04               c88ac1f841b7        2 months ago        70MB
schickling/postgres-backup-s3   latest              e8e41fa54edb        21 months ago       117MB
[lukas@docker-host ~]# docker image prune
WARNING! This will remove all dangling images.
Are you sure you want to continue? [y/N] y
Total reclaimed space: 0B

We see that intermediate layers have got <missing> image id - this id’s was removed because of moving image within different hosts. There are no longer by build cache.

By default commands are truncated but it is possibility to extend them to full size by adding --no-trunc parameter.

Tag images

We can change image name(repository and tag) by docker image tag command:

[lukas@docker-host ~]# docker image ls
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
alpine              latest              a187dde48cd2        2 weeks ago         5.6MB
ubuntu              latest              4e5021d210f6        2 weeks ago         64.2MB

[lukas@docker-host ~]# docker image tag alpine:latest alpine-lukas:1.0

[lukas@docker-host ~]# docker image ls
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
alpine-lukas        1.0                 a187dde48cd2        2 weeks ago         5.6MB
alpine              latest              a187dde48cd2        2 weeks ago         5.6MB
ubuntu              latest              4e5021d210f6        2 weeks ago         64.2MB

In previous example we create another image(it is alias because of smart storage management in Docker).

If we want prepare image for sending remotely we can add repository name, later we can push this image to remote registry:

[lukas@docker-host ~]# docker image tag alpine-lukas:1.0 registry.lukas.it/alpine-lukas:1.0

[lukas@docker-host ~]# docker image ls
REPOSITORY                       TAG                 IMAGE ID            CREATED             SIZE
alpine-lukas                     1.0                 a187dde48cd2        2 weeks ago         5.6MB
alpine                           latest              a187dde48cd2        2 weeks ago         5.6MB
registry.lukas.it/alpine-lukas   1.0                 a187dde48cd2        2 weeks ago         5.6MB
ubuntu                           latest              4e5021d210f6        2 weeks ago         64.2MB
